getObject 方法 (java.lang.String, java.util.Map)

在 Java 程式語言中使用給定的參數名稱，並透過給定的 Map 物件，擷取指定的參數來當做物件。

Microsoft JDBC Driver for SQL Server 目前不支援這個方法。因此，使用這個方法時一定會傳回預設對應。

語法

public java.lang.Object getObject(java.lang.String sCol,
                                  java.util.Map m)

參數

sCol

String，包含參數名稱。

m

Map 物件。

傳回值

Object 值。

例外狀況

SQLServerException

備註

這個 getObject 方法是由 java.sql.CallableStatement 介面中的 getObject 方法所指定。

這個方法將傳回給定資料行的值來當做 Java 物件。此 Java 物件將會是預設的 Java 物件類型，此種類型會對應到資料行的 SQL 類型，並且會對應於 JDBC 規格中所指定的內建類型。如果該值為 SQL NULL，則驅動程式會傳回 Java null。

這個方法也可以用來讀取資料庫特性抽象資料類型。在 JDBC 2.0 API 中，getObject 方法的行為可進行延伸以具體化 SQL 使用者定義類型的資料。如果資料行包含的是結構化或相異的值，這個方法就會產生如同呼叫 getObject(columnIndex, this.getStatement().getConnection().getTypeMap()) 時的行為。

從 SQL Server JDBC 驅動程式 3.0 開始：

  • 將會以 java.sql.Date 物件的形式傳回 date 類型的值。

  • 將會以 java.sql.Time 物件的形式傳回 time 類型的值。

  • 將會以 java.sql.Timestamp 物件的形式傳回 datetime2 類型的值。

  • 將會以 microsoft.sql.DateTimeOffset 物件的形式傳回 datetimeoffset 類型的值。

JDBC 驅動程式中的新功能

Microsoft JDBC Driver 4.0 for SQL Server 包含幾個新功能和增強功能。

有關連接到 SQL Azure 資料庫的資訊

現在有一個主題包含了連接到 SQL Azure 資料庫的相關資訊。如需詳細資訊，請參閱＜連接到 SQL Azure 的資料庫＞。

對於高可用性、災害復原的支援

Microsoft JDBC Driver 4.0 for SQL Server 支援與 SQL Server 2012 中的 AlwaysOn 可用性群組進行高可用性、災害復原連接。如需詳細資訊，請參閱＜JDBC 驅動程式對於高可用性、災害復原的支援＞。

使用 Kerberos 整合式驗證連接到 SQL Server

Microsoft JDBC Driver 4.0 for SQL Server 針對連接到 SQL Server 資料庫的應用程式具有類型 4 Kerberos 整合式驗證。如需詳細資訊，請參閱＜使用 Kerberos 整合式驗證連接到 SQL Server＞。(4.0 之前的 Microsoft JDBC Driver for SQL Server 版本可使用類型 2 Kerberos 整合式驗證)。

存取擴充事件記錄檔中的診斷資訊

您可以存取伺服器擴充事件記錄檔中的資訊，以了解連接失敗。如需詳細資訊，請參閱＜存取擴充事件記錄檔中的診斷資訊＞。

對於疏鬆資料行的額外支援

如果您的應用程式原本存取使用疏鬆資料行之資料表中的資料，您應該會看見效能提升。您可以使用 getColumns 方法 (SQLServerDatabaseMetaData) 來取得資料行的相關資訊 (包括疏鬆資料行資訊)。如需有關 SQL Server 疏鬆資料行的詳細資訊，請參閱＜使用疏鬆資料行＞。

isSparseColumnSet 方法已加入至 SQLServerResultSetMetaData 類別，可指出結果集中的資料行是否為疏鬆資料行集。

Xid.getFormatId

在 Microsoft JDBC Driver 4.0 for SQL Server 之前，JDBC 驅動程式會從應用程式接收格式識別碼，但是不會將該值傳遞至資料庫伺服器。而是，JDBC 驅動程式會將格式識別碼 0 傳遞至資料庫伺服器。

從 Microsoft JDBC Driver 4.0 for SQL Server 開始，JDBC 驅動程式會將格式識別碼從應用程式傳遞至資料庫伺服器。若要取得更新的行為，請務必更新伺服器上的 sqljdbc_xa.dll。如需有關將更新版 sqljdbc_xa.dll 複製到伺服器的詳細資訊，請參閱＜了解 XA 交易＞。

Itanium 不支援 JDBC Driver 4.0 應用程式

Itanium 電腦不支援執行 Microsoft JDBC Driver 4.0 for SQL Server 應用程式。

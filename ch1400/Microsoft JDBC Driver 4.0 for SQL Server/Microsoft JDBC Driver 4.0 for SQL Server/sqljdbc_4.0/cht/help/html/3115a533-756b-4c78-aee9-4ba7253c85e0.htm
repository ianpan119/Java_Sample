SQLServerConnection 成員

下表列出由 SQLServerConnection 類別公開的成員。

建構函式

無。

欄位

名稱

描述

TRANSACTION_SNAPSHOT

用來指定快照集交易隔離等級。

繼承的欄位

類別繼承自：

描述

java.sql.Connection

TRANSACTION_NONE, TRANSACTION_READ_COMMITTED, TRANSACTION_READ_UNCOMMITTED, TRANSACTION_REPEATABLE_READ, TRANSACTION_SERIALIZABLE

方法

名稱

描述

clearWarnings

清除對這個 SQLServerConnection 物件所報告的所有警告。

close

立刻釋放出這個 SQLServerConnection 物件的資料庫和 JDBC 資源，而非等待它們由系統自動釋放。

認可

使從上一次認可或復原後的所有變更成為永久狀態，並且釋放目前由這個 SQLServerConnection 物件保留的任何資料庫鎖定。

createBlob

直接建立 java.sql.Blob 物件，而不使用任何資料。

createClob

直接建立 java.sql.Clob 物件，而不使用任何資料。

createNClob

直接建立 java.sql.NClob 物件，而不使用任何資料。

createStatement

建立 SQLServerStatement 物件，以便將 SQL 陳述式傳送到資料庫。

createSQLXML

直接建立 java.sql.SQLXML 物件，而不使用任何資料。

getAutoCommit

擷取這個 SQLServerConnection 物件的目前自動認可模式。

getCatalog

擷取這個 SQLServerConnection 物件的目前目錄名稱。

getClientConnectionID 方法 (SQLServerConnection)

取得最新連接嘗試的連接識別碼，不論嘗試成功或失敗。

getClientInfo

擷取有關 JDBC 驅動程式所支援之用戶端資訊屬性的資訊。

getHoldability

擷取 SQLServerResultSet 物件的目前保留性，這些物件是使用此 SQLServerConnection 物件所建立的。

getMetaData

擷取 SQLServerDatabaseMetaData 物件，此物件包含此 SQLServerConnection 物件表示連接到其中之資料庫的相關中繼資料。

getTransactionIsolation

擷取這個 SQLServerConnection 物件的目前交易隔離等級。

getTypeMap

擷取與這個 SQLServerConnection 物件相關聯的 Map 物件。

getWarnings

擷取由呼叫這個 SQLServerConnection 物件所報告的第一個警告。

isClosed

指出這個 SQLServerConnection 物件是否已遭關閉。

isReadOnly

指出這個 SQLServerConnection 物件是否處在唯讀模式。

isValid

指出這個 SQLServerConnection 物件是否尚未關閉，而且仍然有效。

nativeSQL

轉換給定的 SQL 陳述式成為資料庫伺服器的原生 SQL 文法。

prepareCall

建立 SQLServerCallableStatement 物件，以便呼叫資料庫預存程序。

prepareStatement

建立 SQLServerPreparedStatement 物件，以便將參數化 SQL 陳述式傳送到資料庫。

releaseSavepoint

從目前的交易，移除指定的 SQLServerSavepoint 物件。

rollback

復原目前交易中所完成的全部變更，並釋放由這個 SQLServerConnection 物件目前所保留的任何資料庫鎖定。

setAutoCommit

設定這個 SQLServerConnection 物件的自動認可模式成為給定的狀態。

setCatalog

設定指定的目錄名稱，以便選取這個 SQLServerConnection 物件之資料庫的子空間，也就是要用來工作的空間。

setClientInfo

設定用戶端資訊屬性的值。

setHoldability

變更 SQLServerResultSet 物件的保留性成為給定的保留性，這些物件是使用此 SQLServerSavepoint 物件所建立的。

setReadOnly

切換這個 SQLServerConnection 物件成為唯讀模式，來當做 JDBC 驅動程式 啟用資料庫最佳化的提示。

setSavepoint

在目前交易中建立未命名儲存點，並傳回表示該儲存點的新 SQLServerSavepoint 物件。

setTransactionIsolation

嘗試變更這個 SQLServerConnection 物件的交易隔離等級成為給定的等級。

setTypeMap

安裝給定的 TypeMap 物件來當做這個 SQLServerConnection 物件的類型對應。

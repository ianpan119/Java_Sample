國家字元集支援

JDBC 驅動程式提供了 JDBC 4.0 API 的支援，其中包含全新的國家字元集轉換 API 方法。這項支援包括 NCHAR、NVARCHAR、LONGNVARCHAR 和 NCLOB JDBC 類型的新 setter、getter 和 updater 方法。

下面是支援國家字元集轉換的新 getter、setter 和 updater 方法清單：

  • SQLServerPreparedStatement：setNString、setNCharacterStream、setNClob。

  • SQLServerCallableStatement：getNClob、getNString、getNCharacterStream、setNString、setNCharacterStream、setNClob。

  • SQLServerResultSet：getNClob、getNString、getNCharacterStream、updateNClob、updateNString、updateNCharacterStream。

若要在應用程式中使用這些方法，您必須將 Classpath 設定為包含 sqljdbc4.jar 檔案。

為了以 Unicode 格式將 String 參數傳送至伺服器，應用程式應該使用新的 JDBC 4.0 國家字元方法，或在使用非國家字元方法時，將 sendStringParametersAsUnicode 連接屬性設定為 "true"。建議的方式為盡可能使用新的 JDBC 4.0 國家字元方法。如需有關 sendStringParametersAsUnicode 連接屬性的詳細資訊，請參閱＜設定連接屬性＞。

以 JDBC 驅動程式執行交易

必須確保其永續性資料一致的所有應用程式，交易處理都會是一項強制需求。利用 Microsoft JDBC Driver for SQL Server，可以在本機或以分散方式執行交易處理。交易的執行為不可部分完成、一致、隔離和持久 (ACID) 的模組。

本節的主題說明 JDBC 驅動程式支援交易的方式，包括隔離等級、交易儲存點和結果集保留性。

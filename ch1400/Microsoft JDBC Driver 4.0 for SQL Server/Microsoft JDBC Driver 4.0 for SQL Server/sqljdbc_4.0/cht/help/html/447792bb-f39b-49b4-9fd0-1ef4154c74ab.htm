JDBC 驅動程式的系統需求

若要使用 Microsoft JDBC Driver for SQL Server 來存取 SQL Server 資料庫的資料，您的電腦上必須安裝下列元件：

  • Microsoft JDBC Driver for SQL Server

  • Java Runtime Environment

Java Runtime Environment 需求

從 Microsoft JDBC Driver for SQL Server 開始，JDBC 驅動程式的 Java Database Connectivity (JDBC) Spec API 支援已經過擴充，可包含 JDBC 4.0 API。JDBC 4.0 導入成 Sun Java SE Development Kit (JDK) 6.0 和 Java Runtime Environment (JRE) 6.0 的一部分。JDBC 4.0 是 JDBC 3.0 的超集。

Microsoft JDBC Driver for SQL Server 同時支援 JDBC 3.0 和 JDBC 4.0。

在 Windows 和 UNIX 作業系統上部署 Microsoft JDBC Driver for SQL Server 時，您必須分別使用安裝套件 sqljdbc_<version>_enu.exe 和 sqljdbc_<version>_enu.tar.gz。如需有關如何部署 JDBC 驅動程式的詳細資訊，請參閱＜部署 JDBC 驅動程式＞主題。

為了支援回溯相容性和可能的升級狀況，JDBC 驅動程式在每個安裝套件中包含了 2 個 JAR 類別庫：sqljdbc.jar 和 sqljdbc4.jar。

JAR

描述

sqljdbc.jar

sqljdbc.jar 類別庫會提供 JDBC 3.0 的支援。

sqljdbc.jar 類別庫需要使用 Java Runtime Environment (JRE) 5.0 版。如果您在 JRE 6.0 上使用 sqljdbc.jar，將會在連接至資料庫時擲回例外狀況。

JDBC Driver 不支援 JRE 1.4。使用 JDBC 驅動程式時，您必須將 JRE 1.4 升級為 JRE 5.0 或 JRE 6.0。在某些情況下，您可能必須重新編譯應用程式，因為它可能與 JDK 5.0 或更新版本不相容。如需詳細資訊，請參閱 Sun Microsystems 網站上的文件。

sqljdbc4.jar

sqljdbc4.jar 類別庫會提供 JDBC 4.0 的支援。它包括 sqljdbc.jar 的所有功能以及新增的 JDBC 4.0 方法。

sqljdbc4.jar 類別庫需要使用 Java Runtime Environment (JRE) 6.0 版。在 JRE 1.4 或 5.0 上使用 sqljdbc4.jar 將會擲回例外狀況。

當您的應用程式必須在 JRE 6.0 上執行時，請使用 sqljdbc4.jar，即使應用程式並未使用 JDBC 4.0 功能也一樣。

請注意，雖然 JDBC Driver 是設計用於與所有主要 Sun 對等 Java 虛擬機器使用並且受該虛擬機器支援，不過它已經在 Sun JRE 5.0 或更新版本上測試過。

SQL Server 需求

JDBC 驅動程式支援 SQL Server 2005 及更新版本的連接。

作業系統需求

JDBC 驅動程式設計為可以在支援使用 Java Virtual Machine (JVM) 的任何作業系統上運作。不過，只有在 Sun Solaris、SUSE Linux、Windows Vista 和 Windows 7 作業系統上正式測試過。

支援的語言

JDBC 驅動程式支援所有 SQL Server 資料行定序。如需有關 JDBC 驅動程式 所支援之定序的詳細資訊，請參閱＜JDBC Driver 的國際功能＞。

如需有關定序的詳細資訊，請參閱《SQL Server 線上叢書》中的＜使用定序＞。

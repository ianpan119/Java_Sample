使用參數中繼資料

若要查詢 SQLServerPreparedStatement 或 SQLServerCallableStatement 物件有關其所包含的參數，Microsoft JDBC Driver for SQL Server 會實作 SQLServerParameterMetaData 類別。此類別包含以單一值格式傳回資訊的許多欄位與方法。

若要建立 SQLServerParameterMetaData 物件，您可使用 SQLServerPreparedStatement 及 SQLServerCallableStatement 類別的 getParameterMetaData 方法。

在下列範例中，對 SQL Server 2005 AdventureWorks 範例資料庫的開啟連接會傳入至函數，並使用 SQLServerCallableStatement 類別的 getParameterMetaData 方法傳回 SQLServerParameterMetaData 物件，然後使用 SQLServerParameterMetaData 物件的各種方法以顯示參數的類型與模式相關資訊，這些參數包含在 HumanResources.uspUpdateEmployeeHireInfo 預存程序中。

public static void getParameterMetaData(Connection con) {
   try {
      CallableStatement cstmt = con.prepareCall("{call HumanResources.uspUpdateEmployeeHireInfo(?, ?, ?, ?, ?)}");
      ParameterMetaData pmd = cstmt.getParameterMetaData();
      int count = pmd.getParameterCount();
      for (int i = 1; i <= count; i++) {
         System.out.println("TYPE: " + pmd.getParameterTypeName(i) + " MODE: " + pmd.getParameterMode(i));
      }
      cstmt.close();
   }
   catch (Exception e) {
      e.printStackTrace();
   }
}

使用含有準備陳述式的 SQLServerParameterMetaData 類別時有一些限制。JDBC 驅動程式支援 SELECT、DELETE、INSERT 與 UPDATE 陳述式。然而，這些陳述式不應該含有子查詢。此外，JDBC 驅動程式也支援使用 IN、IS 及 LIKE 述詞。針對搜尋條件，JDBC 驅動程式支援使用 IS、<、>、LIKE、>=、<=、NOT IN、<>、!>、!<，以及 != 等比較運算子。不支援 FREETEXT 運算子。

setResponseBuffering 方法 (ISQLServerStatement)

設定這個 SQLServerStatement 物件的回應緩衝模式為不區分大小寫的 String full 或 adaptive。

語法

public final void setResponseBuffering(java.lang.String value)

參數

value

String ，包含回應緩衝模式。 有效模式可能是下列其中一個不區分大小寫的字串：full 或 adaptive。

例外狀況

SQLServerException

備註

adaptive 會在必要時指定緩衝處理的可能資料下限。

full 會指定在執行階段從伺服器讀取整個結果。

adaptive 是 JDBC Driver 2.0 和 3.0 版中的預設值，full 是 JDBC Driver 2.0 版之前的預設值。

setResponseBuffering 可讓您覆寫目前 SQLServerStatement 物件的 responseBuffering connection String 屬性。 如需有關使用回應緩衝模式的詳細資訊，請參閱＜使用適應性緩衝＞。

如果應用程式指定給 setResponseBuffering 方法的參數值無效，則會擲回 SQLServerException。

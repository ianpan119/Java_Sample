支援 XML 資料

從 SQL Server 2005 開始，SQL Server 提供了 xml 資料類型，可讓您在 SQL Server 資料庫中儲存 XML 文件和片段。xml 資料類型是 SQL Server 中的內建資料類型，而且與其他內建類型有些相似，例如 int 與 varchar。如同其他內建類型，您可以將 xml 資料類型當做變數類型、參數類型或函數傳回的類型使用；當做建立資料表時的資料行類型；或者在 Transact-SQL CAST 和 CONVERT 函數中使用。在 JDBC 驅動程式 中，xml 資料類型可以對應為字串、位元組陣列、資料流、CLOB、BLOB 或 SQLXML 物件。字串為預設對應。

JDBC 驅動程式提供 JDBC 4.0 API 的支援，此 API 引進 SQLXML 介面。SQLXML 介面會定義與 XML 資料互動和進行操作的方法。SQLXML 是 JDBC 4.0 資料類型，而且它會對應至 SQL Server xml 資料類型。因此，若要在應用程式中使用 SQLXML 資料類型，您必須將 Classpath 設定為包含 sqljdbc4.jar 檔案。如果應用程式在存取 SQLXML 物件及其方法時嘗試使用 sqljdbc3.jar，就會擲回例外狀況。

SQL Server 一定會先驗證 XML 資料，然後再將它儲存於資料庫資料行中。在 SQL Server 2005 或更新版本上執行的應用程式可以使用 SQLXML 資料類型，因為 JDBC 驅動程式會自動將它對應至 xml 資料類型。不過，在 SQL Server 2000 上使用 SQLXML 資料類型將會擲回例外狀況。雖然 sqljdbc4.jar 提供了 SQLXML 支援，不過它只能在 Java Runtime Environment (JRE) 6.0 或更新版本上運作。

本節中的主題將描述 SQLXML 介面以及如何使用 JDBC API 方法來針對 SQLXML 資料類型進行程式設計。
